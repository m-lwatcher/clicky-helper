# Quirky Help — Support / Contact Plan

## Goal
Have a simple, low-maintenance support path that is good enough for first Microsoft Store submission.

## Recommended first-pass support method
**Single support email address** dedicated to Quirky Help.

Why this is the best first move:
- easy to add to Microsoft Store listing
- easy to mention in privacy policy
- low maintenance
- works for bug reports, install issues, and store review questions
- does not require building a support portal right now

## Recommended format
Use one of these:
- `quirkyhelp@<your-domain>`
- `support@<your-domain>`
- `hello@<your-domain>`

Best option if available:
- **`support@` on a simple domain you control**

If no domain is ready yet:
- use a dedicated Gmail/Proton address only for store submission as a temporary bridge
- move to a domain-based address later

## What the support contact needs to do
For first release, support only needs to handle:
- install/setup questions
- provider/API key confusion
- screenshot workflow problems
- copy-command / output-format issues
- store review or account questions

It does **not** need:
- live chat
- ticketing software
- user accounts
- community forum

## Recommended support policy
Keep it light and honest:
- support is best-effort
- no guaranteed response SLA yet
- ask users not to send API keys or sensitive screenshots in plain text
- if they share screenshots/logs manually, those are used only for troubleshooting

## Suggested support text for store listing
**Support contact:**
Contact support at **[support email here]**

## Suggested support text for privacy policy
If you contact support manually, do not send API keys or other secrets in plain text. If you share logs, prompts, or screenshots for troubleshooting, they will be used only to help resolve the issue you reported.
